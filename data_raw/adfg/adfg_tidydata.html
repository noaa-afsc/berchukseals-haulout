Import and Tidy ADFG Supplied Location and Haul-out Data

This article describes the import and tidying of location and haul-out data provided to MML by ADFG for use in manuscript analyzing haul-out behavior of bearded, ribbon, and spotted seals in the Bering and Chukchi seas

Introduction

The focus of this document is not to analyze the data, but instead to document the processing of the raw data (as provided) into a usable data structure for downstream merging with other datasets and analysis procedures. We will focus on reading in the csv files, tidying any data structures or values that need it, and then merging into a single usable tibble.

Read the CSV Files

There are three files provided by ADF&G

  1. ADFG_SealDeploymentTable3.csv
  2. ADFG_SealLocationData3.csv
  3. ADFG_SealTimelineData3.csv

Looks like the only thing we would want to change is the DateTagged column. This is clearly a date-time field, so well create a custom readr::cols function to specify our column types. Note, we will then set the proper time zone to America/Anchorage and then convert to a Date object.

adfg_deploy_cols <- readr::cols(
  DeployIDs = readr::col_character(),
  MapID = readr::col_character(),
  DateTagged = readr::col_datetime(format = "%m/%d/%Y %H:%M"),
  Species = readr::col_character(),
  Age = readr::col_character(),
  Sex = readr::col_character(),
  PrimaryTagType = readr::col_character(),
  FlipperTagType = readr::col_character()
)

adfg_deployments <- readr::read_csv('ADFG_SealDeploymentTable3.csv',
                                    col_types = adfg_deploy_cols) %>% 
  dplyr::mutate(DateTagged = lubridate::force_tz(DateTagged, 
                                                 'America/Anchorage')) %>% 
  dplyr::mutate(DateTagged = lubridate::as_date(DateTagged,

The first author (JML) likes to stick with snake case for column names and variables. So, well do some additional editing of the data to rename columns. While at it, well also convert some of the columns to lower case to limit typo issues down the road.

Another issue we need to tidy up is the way satellite tag type and attachment position is recorded in the dataset. These data are stored within two columns: primary_tag_type and flipper_tag_type. For a proper tidy structure, we need to get these data into two columns: tag_position and tag_type. To do this, well use the tidyr::gather function.

adfg_deployments <- adfg_deployments %>% 
  dplyr::rename(deployid = DeployIDs,
                mapid = MapID,
                date_tagged = DateTagged,
                primary_tag_type = PrimaryTagType,
                flipper_tag_type = FlipperTagType
                ) %>% 
  dplyr::rename_all(tolower) %>% 
  dplyr::mutate(species = tolower(species),
                age = tolower(age),
                sex = tolower(sex)) %>% 
  tidyr::gather(tag_position, tag_type, c(primary_tag_type,flipper_tag_type)) %>% 
  dplyr::mutate(tag_position = ifelse(tag_position == "primary_tag_type",
                   "primary","flipper")) %>% 
  dplyr::filter(!is.na(tag_type)) %>% 
  dplyr::arrange(date_tagged,deployid)

As with the deployment data, the Date column needs to be converted to a proper date-time type. Additionally, we can see that Ptt was brought in as a double when, in fact, this is an integer. There is no way to do the Date or Ptt type conversion via the readr::col_* functions, so we will do a mutate after reading the data in.

adfg_locs_cols <- readr::cols(
  DeployID = readr::col_character(),
  LocationQuality = readr::col_character(),
  Date = readr::col_double(),
  Latitude = readr::col_double(),
  Longitude = readr::col_double(),
  Ptt = readr::col_double(),
  TagType = readr::col_character(),
  MonthNumb = readr::col_character()
)

adfg_locations <- readr::read_csv('ADFG_SealLocationData3.csv',
                                  col_types = adfg_locs_cols) %>% 
  dplyr::mutate(Date = as.Date(Date, origin = "1899-12-30", tz ="UTC") %>% 
                  lubridate::as_datetime(),
                Ptt = as.integer(Ptt))

ADFG Seal Timelines

Last, well go through a similar process with the timeline data.

adfg_timelines <- readr::read_csv('ADFG_SealTimelineData3.csv')

For this dataset, well need to convert the GMTDate to a proper date-time format. And, as before the same with the Ptt as an integer.

adfg_timeline_cols <- cols(
  .default = readr::col_double(),
  DeployIDs = readr::col_character(),
  TagType = readr::col_character(),
  HistType = readr::col_character(),
  MonthNumb = readr::col_character(),
  LocationQuality = readr::col_character(),
  Sum = readr::col_character()
)

adfg_timelines <- readr::read_csv('ADFG_SealTimelineData3.csv',
                                  col_types = adfg_timeline_cols) %>% 
  dplyr::mutate(GMTDate = as.Date(GMTDate, origin = "1899-12-30", 
                                     tz = "UTC") %>% 
                  lubridate::as_datetime(),
                Ptt = as.integer(Ptt),
                TagType = toupper(TagType)) %>% 
  dplyr::rename(deployid = DeployIDs,
                tag_type = TagType,
                hist_type = HistType,
                date_time = GMTDate) %>% 
  dplyr::rename_all(tolower) %>% 
  dplyr::select(-c(monthnumb,latitude,longitude,locationquality,sum,hist_type))

We need to do some additional processing of the timeline data to work with the odd non-tidy data structure. In this case, each bin* column represents an hour of each day.

bins <- list(bin = paste("bin", 1:24, sep=""), hour=0:23) %>% as_tibble()

adfg_timelines <- adfg_timelines %>% 
        tidyr::gather(bin, percent_dry, starts_with('bin')) %>%
        dplyr::left_join(bins, by = "bin") %>%
        dplyr::mutate(date_time = date_time + lubridate::hours(hour),
                      data_date = lubridate::floor_date(date_time,"day")) %>%
        dplyr::select(c(deployid,ptt,tag_type,date_time,data_date,percent_dry)) %>% 
        dplyr::arrange(deployid,date_time)
